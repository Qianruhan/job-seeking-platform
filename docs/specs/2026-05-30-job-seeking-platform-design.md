# 求职准备平台 — 产品设计文档

**版本**: v1.0  
**日期**: 2026-05-30  
**作者**: ZhuanZ  
**定位**: 面向转行/零基础求职者的 AI 求职准备平台

---

## 1. 产品概述

### 1.1 一句话定位

覆盖"看清市场 → 补齐能力 → 包装简历 → 面试实战"全链路的 AI 求职准备平台。

### 1.2 目标用户

- **主要用户**: 转行求职者、零基础求职者
- **次要用户**: 有经验的求职者（针对性准备特定公司/岗位）
- **目标岗位类型**: 计算机类（研发、产品、算法），MVP 聚焦后端开发、AI 产品经理、算法工程师

### 1.3 解决的核心痛点

| 痛点 | 解决方案 |
|------|---------|
| 转行/零基础不知道从何学起，经验帖多而杂抓不住重点 | JD 分析 + 面经 + 公开学习路线交叉验证，生成清晰的能力画像和学习路线 |
| 一份简历海投，不针对岗位定制 | 选定 JD 后 AI 分析匹配度，给出逐条改写建议 |
| 面试前需要反复搜索不同平台的面经 | 多平台面经统一搜索，AI 摘要提炼 |
| 缺乏面试经验，第一次面试紧张 | AI 模拟面试（通用 + 公司定制），面试后评分和改进建议 |
| 不知道目标公司/岗位的具体面试风格 | 基于该公司面经库生成定制面试，还原真实面试风格 |

### 1.4 产品设计原则

- **AI 辅助而非取代**: 传统 UI 为主交互，AI 在幕后做分析和生成
- **不强套 Agent**: Agent 仅用于需要多步推理的串联场景（能力画像、简历诊断、模拟面试），单步任务走普通 LLM 调用
- **数据可溯源**: 所有爬取内容展示来源链接、爬取时间
- **闭环驱动**: 学习路线 → 简历包装 → 模拟面试 → 反馈回学习路线

## 2. 用户旅程

```
注册/登录
    │
    ▼
设置目标岗位 + 输入当前背景
    │
    ▼
全局诊断（首页仪表盘）
    │
    ├──→ 阶段一：岗位洞察
    │     ├── 市场全景（技能词云、业务分布、JD 列表）
    │     └── 能力画像 + 学习路线（我的技能 vs 市场要求、差距清单、学习路径）
    │
    ├──→ 阶段二：简历包装
    │     上传简历 + 选择目标 JD → 匹配分析 → 改写建议 → 在线编辑 → 保存版本
    │
    ├──→ 阶段三：面经知识库
    │     按公司/岗位/时间/平台搜索 → AI 摘要
    │
    └──→ 阶段四：模拟面试
          通用面试（标准流程）or 具体公司面试（基于面经库定制）
          → 多轮对话 → 评分报告 → 薄弱项同步学习路线
```

用户可在任意阶段进入，但系统会在仪表盘给出"下一步建议"引导。

│  │              │ │              │ │           ││
│  │ JD 解析      │ │ 匹配打分      │ │ 出题+追问  ││
│  │ 能力提炼     │ │ 差距诊断      │ │ 评分+反馈  ││

### 3.1 技术栈

| 层 | 技术选型 | 说明 |
|----|---------|------|
| 前端框架 | React + TypeScript | 生态广，AI 相关组件丰富 |
| UI 组件库 | Ant Design / Shadcn | 根据后续 UI 风格确定 |
| 后端框架 | FastAPI (Python) | 高性能异步，Agent 开发友好 |
| Agent 编排 | LangGraph | 状态机模型，多步推理可控 |
| 主数据库 | PostgreSQL + pgvector | 关系 + 向量检索一体，免额外部署 |
| 缓存/队列 | Redis | 爬取任务队列 + 热数据缓存 |
| 定时调度 | APScheduler | Python 原生，配置文件驱动 |
| LLM 提供商 | DeepSeek / OpenAI 可切换 | 配置驱动，Agent 层切换无代码改动 |

### 3.2 Agent 使用边界

**使用 Agent 的场景**（需要多步推理串联）:
- 能力画像 Agent: 多份 JD 提炼共性 → 对比用户背景 → 生成学习路线
- 简历诊断 Agent: 匹配度评估 → 差距诊断 → 改写建议
- 面试官 Agent: 初始化角色 → 多轮出题追问 → 评分反馈

**不使用 Agent 的场景**（单步任务）:
- 面经搜索: 向量检索 + 关键词过滤 + LLM 摘要（一个函数完成）
- JD 分析: 单次 LLM 调用即可结构化输出
- 爬取动态展示: 纯数据库查询

## 4. 核心模块设计

### 4.1 岗位洞察

**子模块一：市场全景**
- 筛选栏: 岗位类型（下拉）、薪资范围（滑块）、城市
- 结果展示: 技能词云图、业务领域分布图、JD 摘要列表
- JD 详情: 原始文本、来源链接、爬取时间

**子模块二：能力画像 + 学习路线**
- 用户填写当前背景（技能标签、学历、工作年限）
- Agent 执行三步: 市场扫描 → 差距对比 → 生成学习路线
- 展示: 技能雷达图、差距清单（优先级排序）、学习路线时间线
- 数据来源: JD 分析结果 + 面经库 + 公开学习路线帖（知乎/掘金）
- 学习路线每项包含: 学什么 → 达到什么程度 → 推荐资源

### 4.2 简历包装

- 上传简历（支持 PDF/Word）
- 选择目标 JD（从岗位洞察库选或手动粘贴）
- Agent 执行: 简历解析 → 匹配打分（硬技能/软素质/业务相关性）→ 逐条改写建议 → 硬伤提醒
- 在线编辑 + 多版本存档（每个 JD 对应一个简历版本）
- 输出: 匹配度报告（分数 + 维度拆解）、改写建议、硬伤清单

### 4.3 面经知识库

- 搜索 + 筛选: 公司、岗位、时间范围、来源平台
- 结果: 面经卡片列表（公司 | 岗位 | 标签 | 时间 | 来源链接）
- 点击展开: AI 摘要 + 原文链接
- 数据来源: 牛客网、力扣、小红书（尽力而为）

### 4.4 模拟面试

- **通用模式**: 基于岗位的标准化面试流程（自我介绍 → 基础知识 → 项目深挖 → 场景题 → 反问）
- **具体公司模式**: Agent 读取该公司面经库，提取面试风格和题型偏好，生成定制面试官角色
- 面试过程: 聊天式对话，Agent 扮演面试官，根据回答质量决定追问或下一题
- 面试后: 评分报告（技术深度、表达清晰度、项目理解、临场反应）+ 2-3 个具体改进点
- 薄弱项自动同步到学习路线

## 5. Agent 工作流

### 5.1 能力画像 Agent

```
输入: 目标岗位 + 用户当前背景
流程:
  Step 1 — 市场扫描
    查询该岗位所有 JD 分析结果和面经 → 提炼共性要求
    → 输出"通用能力模型"
  Step 2 — 差距对比
    用户技能 vs 通用能力模型 → 分类: 已具备 / 部分具备 / 完全缺失
  Step 3 — 生成学习路线
    按优先级排序，结合公开学习路线帖交叉验证
    每项: 学什么 → 面试标准 → 推荐资源
输出: 雷达图数据 + 差距清单 + 学习路线
```

### 5.2 简历诊断 Agent

```
输入: 简历原文 + JD 原文
流程:
  Step 1 — 解析提取
    简历: 技能标签、项目经历关键词
    JD: 要求标签、业务关键词
  Step 2 — 匹配打分
    硬技能匹配度、软素质匹配度、业务相关性 → 综合评分
  Step 3 — 改写建议
    针对 JD 关键词嵌入、措辞优化
  Step 4 — 硬伤提醒
    简历完全未提及的 JD 核心要求
输出: 匹配度报告 + 逐条改写建议 + 硬伤清单
```

### 5.3 面试官 Agent

```
输入: 面试类型 + 目标 JD（可选）+ 关联面经
流程:
  Step 1 — 初始化角色
    通用模式: 标准化面试模板
    具体模式: 读面经库 → 提取面试风格和题型偏好
    → 生成 system prompt
  Step 2 — 多轮面试
    自我介绍 → 基础知识 → 项目深挖 → 场景题 → 反问
    每轮根据回答质量决定追问、提示或进入下一题
  Step 3 — 评分反馈
    各维度打分 → 指出改进点 → 薄弱项标签同步学习路线
输出: 对话记录 + 评分报告 + 改进建议
```

## 8. 工程健壮性

### 8.1 配置驱动

所有可变参数通过 YAML 配置文件管理，不硬编码:

```yaml
crawl:
  boss_zhipin:
    interval_minutes: 360
    enabled: true
  xiaohongshu:
    interval_minutes: 720
    enabled: false  # 反爬严格时可关闭

llm:
  provider: deepseek
  model: deepseek-chat
  max_tokens: 4096

agent:
  retry_count: 3
  timeout_seconds: 120
  trace_enabled: true
```

### 8.2 Agent 追踪

每次 Agent 调用记录:
- 输入（截断存储）
- 输出（结构化 JSON）
- 每步耗时
- Token 用量
- 错误/重试记录

Dashboard 可查看 Agent Trace 历史，便于调试和优化。

### 8.3 爬取容错

- 每个平台独立 Worker，单个失败不影响其他
- 爬取日志全量记录（成功数、失败数、耗时、错误堆栈）
- 去重策略: URL 哈希判重，幂等入库
- 失败重试: 指数退避 + 最大重试次数

### 8.4 安全性

- 密码 bcrypt 哈希存储
- JWT Token 认证
- 用户数据隔离（所有查询加 user_id 过滤）
- API 输入校验（简历上传、JD 粘贴等用户输入）

## 10. MVP 范围

### 10.1 MVP 内容

| 模块 | 范围 | 不做 |
|------|------|------|
| 用户系统 | 注册、登录、个人背景设置 | 第三方登录、密码找回 |
| 岗位洞察 | 后端/算法/AI产品 3 类岗位 | 全岗位覆盖 |
| 简历包装 | 上传 + 匹配分析 + 改写建议 + 在线编辑 + 多版本 | — |
| 面经知识库 | 按公司/岗位搜索 + AI 摘要 | 社区评论、点赞 |
| 模拟面试 | 通用模式 + 具体公司定制模式 | — |
| 爬取仪表盘 | 最近一天爬取动态 + 溯源 | 爬取配置后台 |
| 数据来源 | BOSS直聘、牛客、力扣、知乎、掘金、小红书（尽力） | — |

### 10.2 MVP 验收标准

用户可完成端到端流程: 注册 → 设定目标岗位 → 查看岗位画像和学习路线 → 上传简历做匹配 → 搜索面经 → 完成一次模拟面试 → 查看评分报告

### 10.3 后续迭代（V2+）

- 更多岗位类型覆盖
- 面经社区功能（用户补充面经、点赞）
- 学习路线进度追踪
- 更多数据源接入
- 求职进度管理（投递记录、面试日历）
- AI 职业规划师（多岗位对比 + 长期路径）
- 微信小程序

## 12. 部署

- **目标**: 部署到云服务器（阿里云/腾讯云），生成公网 URL
- **服务组件**: FastAPI + PostgreSQL + Redis + React 前端
- **部署方式**: Docker Compose 编排（可选 Nginx 反向代理）
- **CI/CD**: 非 MVP 必须，可后续补充
